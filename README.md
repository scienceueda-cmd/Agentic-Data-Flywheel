# 🚀 Agentic Data Flywheel for Open-WebUI

このプロジェクトは、ローカルPC上で完全自律型のAIエージェントを構築するための**「データフライホイール＆ファインチューニング環境」**です。
Open-WebUIの日常のチャット履歴から学習データを抽出し、Llama 3などの軽量ローカルモデルに「自律エージェントとしての思考（ReActフォーマット）」を焼き付け、LangChainで実行します。

## 🌟 アーキテクチャ（システム構成）

本システムは、以下の3つの主要コンポーネントが円環（フライホイール）となって動作します。

1. **データ収集＆合成エンジン (`src/log_monitor.py`)**
   - Open-WebUIのSQLiteデータベース（`webui.db`）を常時監視します。
   - 【重要】抽出された生ログは、そのままでは学習に使えません。**「マスターAI（GPT-4やGemini等）」**に渡し、「エージェントとして振る舞うべき理想のトラジェクトリ（ReAct形式）」へとリライト（合成）させることで、高品質な `lora_dataset.jsonl` を作成します（Knowledge Distillation）。
2. **LoRAファインチューニング・エンジン (`src/train_lora.py`)**
   - 蓄積された `lora_dataset.jsonl` を用いて、`unsloth` ライブラリの力でベースモデル（例: Llama-3 8B）を高速・省メモリ（VRAM 8GB対応）でファインチューニングします。
   - ツールの使い方（フォーマット）をモデルの「筋肉」に直接焼き付けます。
3. **エージェント実行エンジン (`src/run_agent.py`)**
   - 学習済みのLoRAアダプター（`lora_model`）をベースモデルに結合します。
   - LangChainの `AgentExecutor` を用い、実際にターミナルコマンド（`run_command`）やファイル読み込み（`view_file`）、ウェブ検索（`web_search`）を実行する自律ループを回します。
   - `.agents/skills/` フォルダのMarkdownを動的に読み込む**RAG（スキルシステム）**と、コマンド実行前の**Human-in-the-Loop（Y/N確認）**の安全装置を備えています。

## 💻 動作環境（システム要件）
ローカルPCで「モデルの学習」と「推論」を行うため、以下のスペックが必要です。
- **OS**: Windows 10 / 11
- **GPU**: NVIDIA製グラフィックボード（**VRAM 8GB以上必須**。例: RTX 2070 SUPER, RTX 3060, RTX 4060等）
- **RAM**: 16GB以上推奨
- **Python**: 3.12
- **ストレージ**: 10GB以上の空き容量（ベースモデルのダウンロード用）

## 📦 インストールと準備（前提条件）

**【必須条件】**
このシステムは、あなたのPCにすでに **Open-WebUI** がインストールされていることを前提とした「拡張パック」です。まだOpen-WebUIをお持ちでない場合は、先に公式（https://github.com/open-webui/open-webui）からインストールを済ませてください。

### セットアップ手順

`Setup.bat` をダブルクリックするだけで、以下の処理が全自動で行われます。

1. 必要なPythonライブラリのインストール（`requirements.txt`）
2. Windows起動時に自動でログ監視を始める「スタートアップ登録」

*(注意: 初回学習時にHuggingFaceからベースモデルがダウンロードされます。)*

## 🎮 使い方

プロジェクト直下にある **`Agent_Control_Panel.bat`** をダブルクリックしてコントロールパネルを開きます。

- **[1] 監視エージェントの起動**: 
  裏側（バックグラウンド）でサイレント起動し、Open-WebUIのログを監視して学習データを蓄積し続けます。（※バッチが作成されたら、マスターAIに渡してJSONL化してください）
- **[2] 脳みその学習を開始**: 
  蓄積されたデータを使ってファインチューニング（LoRA学習）を行います。
- **[3] エージェントを起動**: 
  学習済みの脳みそを使って、ターミナル上で自律エージェント（CLI）を起動し、指示を与えます。（※実際のコマンド実行テスト用）
- **[4] 必須ライブラリのインストール**:
  初回環境構築やアップデート時に `requirements.txt` を適用します。
- **[5] WebUI連携用 APIサーバー起動 (FastAPI Server)**: 
  学習済みのモデルをバックグラウンドの「OpenAI互換APIサーバー」として起動します。Open WebUIからこのエージェントと直接チャットができるようになります。

### Open WebUI との連携方法
1. コントロールパネルで `[5]` を実行し、APIサーバーを起動させたままにします。（`http://localhost:8000/v1` で待機します）
2. Open WebUI の画面を開き、左下の「設定 (Settings)」＞「接続 (Connections)」を開きます。
3. `OpenAI API` の項目で「＋」ボタンを押し、以下のように設定します：
   - **URL**: `http://localhost:8000/v1`
   - **API Key**: `sk-1234` （※ダミーでOKです）
4. 右下の保存マークを押して「接続が成功しました」と出れば連携完了です！
5. メイン画面のモデル選択プルダウンに `antigravity-agent` が追加されるので、それを選択して話しかけてみてください！

## ⚙️ モデルとキャッシュの変更（上級者向け）
デフォルトでは `unsloth/llama-3-8b-Instruct-bnb-4bit` がダウンロードされて使用されますが、別のモデルを使いたい場合や、キャッシュの保存先（Cドライブの圧迫回避など）を変更したい場合は、プロジェクト直下の **`config.json`** を書き換えてください。

```json
{
    "base_model_name": "C:\\my_models\\llama-3-local",
    "hf_cache_dir": "D:\\my_large_drive\\hf_cache"
}
```
`base_model_name`: 使用するモデルの指定（ローカルの絶対パスも可能）
`hf_cache_dir`: ネットからモデルをダウンロードした際の一時保存先（デフォルトは相対パス `./hf_cache` なのでUSBメモリ等に入れてもポータブルに動きます）

書き換えるだけで、学習時も実行時も自動的にその設定が反映されるようになります。

## 🛠️ スキル（カンペ）の追加方法
このエージェントはRAGシステムを搭載しています。
プロジェクト内の `.agents/skills/` フォルダに任意のMarkdownファイル（例: `git_rules.md` や `company_guidelines.md`）を配置するだけで、エージェントが起動時に自動で読み込み、自身のプロンプトとして吸収します。
